# 台股 AI 自動化監控系統 — 完整教學

## 系統簡介

本系統以 **Google Apps Script** 為核心，串接 **Gemini AI** 與 **Telegram Bot**，打造一套全自動的台股追蹤、分析與推播機器人。

---

## 架構總覽

```
Google Sheets
    ├── 每日行情    ← 自動抓取台股收盤資料
    ├── 追蹤清單    ← 個人自選股設定
    ├── 國際盤勢    ← 手動輸入美股指數
    └── AI紀錄      ← 所有 AI 報告存檔

Google Apps Script (5 個齒輪)
    ├── 齒輪一：fetchStockData()      每日抓取收盤
    ├── 齒輪二：nightlyReview()       夜間覆盤報告
    ├── 齒輪三：morningForecast()     晨間作戰計畫
    ├── 齒輪四：generateAnalysisDoc() 深度分析文案
    └── 齒輪五：scanPotentialStocks() 持股健檢掃描

外部 API
    ├── 證交所 TWSE OpenAPI  ← 股價來源 (免費)
    ├── Gemini API           ← AI 分析引擎
    └── Telegram Bot API     ← 推播通知
```

---

## 事前準備

### 1. 建立 Telegram Bot

1. 在 Telegram 搜尋 `@BotFather`
2. 輸入 `/newbot`，依指示設定名稱與 username
3. 取得 **Bot Token**（格式：`1234567890:ABCDefgh...`）
4. 將 Bot 加入你的頻道或群組，取得 **Chat ID**
   - 傳訊息給 Bot 後，開啟 `https://api.telegram.org/bot<TOKEN>/getUpdates` 查看 `chat.id`

### 2. 申請 Gemini API 金鑰

1. 前往 [Google AI Studio](https://aistudio.google.com/)
2. 點選「Get API Key」→「Create API Key」
3. 複製金鑰備用

### 3. 建立 Google Sheets

建立一個新試算表，新增以下四個分頁（工作表名稱需完全一致）：

| 分頁名稱 | 用途 | 欄位說明 |
|---------|------|---------|
| `每日行情` | 儲存每日收盤資料 | A:日期、B:代號、C:名稱、D:收盤價、E:漲跌、F:成交量 |
| `追蹤清單` | 個人自選股清單 | A:代號、B:名稱、C:支撐價(自動填)、D:壓力價(自動填)、...H:隔日策略(自動填) |
| `國際盤勢` | 手動輸入美股 | A:指數名稱、B:漲跌幅 (A2:B4，共3筆) |
| `AI紀錄` | AI 報告存檔 | A:時間、B:類型、C:內容 |

## 部署步驟

### Step 1：開啟 Apps Script 編輯器

在 Google Sheets 點選「擴充功能」→「Apps Script」

### Step 2：儲存 API 金鑰到保險箱

> 金鑰**絕對不能**直接寫在程式碼裡，必須存入 Script Properties。

在 Apps Script 編輯器左側點選「專案設定」→「指令碼屬性」，新增三筆屬性：

| 屬性名稱 | 值 |
|---------|-----|
| `TG_TOKEN` | 你的 Telegram Bot Token |
| `TG_CHAT_ID` | 你的 Telegram Chat ID |
| `GEMINI_KEY` | 你的 Gemini API 金鑰 |

### Step 3：貼上程式碼

將完整程式碼貼入編輯器，存檔（Ctrl+S）。

### Step 4：設定自動觸發排程

點選左側「觸發條件」→「新增觸發條件」，依下表設定：

| 函式 | 觸發類型 | 建議時間 |
|------|---------|---------|
| `fetchStockData` | 以時間為基礎 → 日計時器 | 下午 1:30（收盤後） |
| `nightlyReview` | 以時間為基礎 → 日計時器 | 晚上 9:00 |
| `morningForecast` | 以時間為基礎 → 日計時器 | 早上 8:00 |
| `scanPotentialStocks` | 以時間為基礎 → 日計時器 | 晚上 8:00 |

## 五大齒輪詳解

### 齒輪一：`fetchStockData()` — 每日抓取收盤

**資料來源**：證交所 TWSE OpenAPI（免費、無需金鑰）

```
https://openapi.twse.com.tw/v1/exchangeReport/STOCK_DAY_ALL
```

**核心邏輯**：
1. 抓取全市場所有股票當日收盤資料（JSON 格式）
2. 過濾出 `myStocks` 陣列中的自選股代號
3. 將日期、代號、名稱、收盤價、漲跌、成交量寫入「每日行情」分頁

**自訂自選股**：修改 `myStocks` 陣列即可：
```javascript
var myStocks = ["2330", "2317", "0050"]; // 填入你要追蹤的股票代號
```

---

### 齒輪二：`nightlyReview()` — 夜間覆盤報告

**執行時機**：盤後（建議晚上 9 點）

**流程**：
1. 從「每日行情」讀取當日最新資料
2. 整理成「股名+漲跌」的摘要字串
3. 送入 Gemini，要求生成 **150 字內** 的純文字覆盤報告
4. 報告存入「AI紀錄」並推播至 Telegram

**Prompt 設計重點**：
- 角色設定：「頂尖台股操盤手」
- 輸出格式：要求純文字，禁用 Markdown（避免 Telegram 顯示亂碼）
- 內容要求：整體盤勢簡評 + 隔日建議

---

### 齒輪三：`morningForecast()` — 晨間作戰計畫

**執行時機**：開盤前（建議早上 8 點）

**資料來源**：「國際盤勢」分頁的 `A2:B4`（需手動輸入前一晚美股數據）

**Prompt 涵蓋 15 大分析面向**：
1. 市場熱度與量價結構
2. 技術面支撐與壓力
3. 操作策略與風控
4. 近期催化劑與產業速覽
5. 股價位階與大盤強弱對比
6. 主力與三大法人籌碼
7. 法說會、營收、除權息等重大事件
8. 量價型態與均線位階
9. 投信外資籌碼研判
10. 實戰劇本與進場情境
11. 嚴格停損條件
12. 同族群題材連動分析
13. 分批布局區間或突破觸發點
14. 進場邏輯說明
15. 損益比（R/R）試算

---

### 齒輪四：`generateAnalysisDoc()` — 深度分析文案

**用途**：針對自選股生成較完整的書面分析報告

**與齒輪二的差異**：
- 齒輪二：快速覆盤，150 字，給結論
- 齒輪四：完整分析，含明日風險提示，存入「AI紀錄」備查

---

### 齒輪五：`scanPotentialStocks()` — 盤後持股健檢（核心功能）

**最強大的齒輪**，具備以下特點：

#### 自帶金鑰機制
在函式內部重新讀取一次 Script Properties，防止全域變數失效導致 `not defined` 錯誤。

#### 動態讀取追蹤清單
自動從「追蹤清單」分頁抓取所有股票代號與名稱，支援 `TPE:` 前綴格式（TradingView 格式）。

#### 四大分析板塊 Prompt
| 板塊 | 內容 |
|------|------|
| 一、市場位階與動能 | 熱度、相對強弱、均線型態 |
| 二、籌碼與基本面催化 | 法人動向、催化劑、族群效應 |
| 三、實戰劇本與進場策略 | 支撐壓力、進場劇本、觸發價位 |
| 四、嚴格風控與期望值 | 停損、減碼、R/R 損益比試算 |

#### 結構化資料解析（DATA 標籤）
Prompt 要求 AI 在報告末尾輸出特定格式：
```
[DATA:代號|支撐價|壓力價|隔日策略]
範例：[DATA:2330|1050|1120|量縮回測支撐不破可低接]
```

程式以正規表達式解析後，**自動回填**至「追蹤清單」對應欄位：
- C 欄 ← 支撐價
- D 欄 ← 壓力價
- H 欄 ← 隔日策略

#### 自動重試機制
最多重試 3 次，每次間隔 3 秒，應對 Gemini API 503 壅塞。

## 安全設計說明

### 為何不把金鑰寫在程式碼裡？

| 做法 | 風險 |
|------|------|
| 直接寫在程式碼 | 若分享給他人或公開專案，金鑰立即外洩 |
| 使用 Script Properties | 金鑰加密儲存，程式碼可安全分享 |

### Script Properties 讀取方式
```javascript
var scriptProperties = PropertiesService.getScriptProperties();
var myKey = scriptProperties.getProperty('MY_KEY_NAME');
```

## 常見問題排解

### Q1：Telegram 沒有收到訊息
- 確認 Bot Token 與 Chat ID 正確
- 確認 Bot 已被加入聊天室且未被封鎖
- 檢查 Apps Script 執行紀錄是否有錯誤（左側「執行項目」）

### Q2：Gemini 回傳 503 錯誤
- 為正常現象，系統會自動重試 3 次
- 若持續失敗，表示 API 用量超過免費額度上限
- 確認 `gemini-flash-latest` 模型名稱是否正確

### Q3：`fetchStockData` 沒有寫入資料
- 確認工作表名稱為「每日行情」（含中文，全形符號）
- 確認 `myStocks` 陣列內的代號為字串格式
- 注意：TWSE API 只在交易日有資料，假日執行會得到空回應

### Q4：`scanPotentialStocks` 找不到追蹤清單
- 確認工作表名稱為「追蹤清單」
- 確認第一列為標題列，資料從第二列開始
- 代號欄支援 `2330` 或 `TPE:2330` 兩種格式

## 擴充建議

### 加入停損警報
可新增一個函式，掃描「追蹤清單」的支撐價，若收盤價跌破支撐即推播警報：
```javascript
function alertBelowSupport() {
  // 比對 C 欄支撐價 vs D 欄最新收盤，跌破即發送 Telegram 警告
}
```

### 加入週報功能
每週五新增一個 `weeklyReport()` 函式，統計當週漲跌幅排行並生成週報。

### 擴充國際盤勢自動化
目前「國際盤勢」需手動輸入，可串接 Yahoo Finance 非官方 API 自動抓取美股三大指數。
