# 台股 AI 自動化監控系統 — 完整教學

## 系統簡介

本系統以 **Google Apps Script** 為核心，串接 **Gemini AI** 與 **Telegram Bot**，打造一套全自動的台股追蹤、分析與推播機器人。

---

## 架構總覽

```
Google Sheets
    ├── 每日行情    ← 自動抓取台股收盤資料
    ├── 追蹤清單    ← 個人自選股設定
    ├── 國際盤勢    ← 手動輸入美股指數
    └── AI紀錄      ← 所有 AI 報告存檔

Google Apps Script (5 個齒輪)
    ├── 齒輪一：fetchStockData()      每日抓取收盤
    ├── 齒輪二：nightlyReview()       夜間覆盤報告
    ├── 齒輪三：morningForecast()     晨間作戰計畫
    ├── 齒輪四：generateAnalysisDoc() 深度分析文案
    └── 齒輪五：scanPotentialStocks() 持股健檢掃描

外部 API
    ├── 證交所 TWSE OpenAPI  ← 股價來源 (免費)
    ├── Gemini API           ← AI 分析引擎
    └── Telegram Bot API     ← 推播通知
```

## 事前準備

### 1. 建立 Telegram Bot

1. 在 Telegram 搜尋 `@BotFather`
2. 輸入 `/newbot`，依指示設定名稱與 username
3. 取得 **Bot Token**（格式：`1234567890:ABCDefgh...`）
4. 將 Bot 加入你的頻道或群組，取得 **Chat ID**
   - 傳訊息給 Bot 後，開啟 `https://api.telegram.org/bot<TOKEN>/getUpdates` 查看 `chat.id`

### 2. 申請 Gemini API 金鑰

1. 前往 [Google AI Studio](https://aistudio.google.com/)
2. 點選「Get API Key」→「Create API Key」
3. 複製金鑰備用

### 3. 建立 Google Sheets

建立一個新試算表，新增以下四個分頁（工作表名稱需完全一致）：

| 分頁名稱 | 用途 | 欄位說明 |
|---------|------|---------|
| `每日行情` | 儲存每日收盤資料 | A:日期、B:代號、C:名稱、D:收盤價、E:漲跌、F:成交量 |
| `追蹤清單` | 個人自選股清單 | A:代號、B:名稱、C:支撐價(自動填)、D:壓力價(自動填)、...H:隔日策略(自動填) |
| `國際盤勢` | 手動輸入美股 | A:指數名稱、B:漲跌幅 (A2:B4，共3筆) |
| `AI紀錄` | AI 報告存檔 | A:時間、B:類型、C:內容 |

## 部署步驟

### Step 1：開啟 Apps Script 編輯器

在 Google Sheets 點選「擴充功能」→「Apps Script」

### Step 2：儲存 API 金鑰到保險箱

> 金鑰**絕對不能**直接寫在程式碼裡，必須存入 Script Properties。

在 Apps Script 編輯器左側點選「專案設定」→「指令碼屬性」，新增三筆屬性：

| 屬性名稱 | 值 |
|---------|-----|
| `TG_TOKEN` | 你的 Telegram Bot Token |
| `TG_CHAT_ID` | 你的 Telegram Chat ID |
| `GEMINI_KEY` | 你的 Gemini API 金鑰 |

### Step 3：貼上程式碼

將下方完整程式碼貼入編輯器，存檔（Ctrl+S）。

### Step 4：設定自動觸發排程

點選左側「觸發條件」→「新增觸發條件」，依下表設定：

| 函式 | 觸發類型 | 建議時間 |
|------|---------|---------|
| `fetchStockData` | 以時間為基礎 → 日計時器 | 下午 1:30（收盤後） |
| `nightlyReview` | 以時間為基礎 → 日計時器 | 晚上 9:00 |
| `morningForecast` | 以時間為基礎 → 日計時器 | 早上 8:00 |
| `scanPotentialStocks` | 以時間為基礎 → 日計時器 | 晚上 8:00 |

```javascript
// ⚙️ 齒輪一：每日自動抓取收盤資料
// ==========================================
function fetchStockData() {
  var url = "https://openapi.twse.com.tw/v1/exchangeReport/STOCK_DAY_ALL";
  var response = UrlFetchApp.fetch(url);
  var data = JSON.parse(response.getContentText());
  
  var myStocks = ["7721","4956","2388","3131","3081","3324","3017","4979","1513","1519",
                  "2308","2317","2454","3711","2382","2881","2383",
                  "2882","2891","6752","8473","6667","3029","6806"];
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("每日行情");
  var today = Utilities.formatDate(new Date(), "GMT+8", "yyyy-MM-dd");

  data.forEach(function(stock) {
    if (myStocks.indexOf(stock.Code) !== -1) {
      sheet.appendRow([
        today, 
        stock.Code, 
        stock.Name, 
        stock.ClosingPrice, 
        stock.Change, 
        stock.TradeVolume
      ]);
    }
  });
}

// 🌙 齒輪二：夜間覆盤報告
// ==========================================
function nightlyReview() {
  var sheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("每日行情");
  var allData = sheet.getDataRange().getValues();
  if (allData.length < 2) return;
  
  var lastDateRaw = allData[allData.length - 1][0];
  var lastDateStr = new Date(lastDateRaw).toDateString();
  var todayData = allData.filter(row => new Date(row[0]).toDateString() === lastDateStr);
  
  var stockSummary = todayData.map(row => row[2] + "漲跌:" + row[4]).join("、");
  var date = Utilities.formatDate(new Date(lastDateRaw), "GMT+8", "yyyy-MM-dd");

  var prompt = "你是頂尖台股操盤手。請根據以下今日自選股數據，以淺顯易懂的方式，寫 150 字內的綜合覆盤報告：日期 " + date + "。表現：" + stockSummary + "。報告需含：1.整體盤勢與強弱族群簡評 2.隔日總體建議。回覆內容請一律使用『純文字』，絕對不要使用任何 Markdown 格式符號。";

  var geminiUrl = "https://generativelanguage.googleapis.com/v1beta/models/gemini-flash-latest:generateContent?key=" + gemini_apiKey;
  var payload = { "contents": [{ "parts": [{ "text": prompt }] }] };
  
  try {
    var response = UrlFetchApp.fetch(geminiUrl, {
      "method": "post", "contentType": "application/json", "payload": JSON.stringify(payload)
    });
    var aiResult = JSON.parse(response.getContentText());
    var aiReport = aiResult.candidates[0].content.parts[0].text;

    var logSheet = SpreadsheetApp.getActiveSpreadsheet().getSheetByName("AI紀錄");
    logSheet.appendRow([new Date(), "夜間覆盤", aiReport]);

    var message = "📊 夜間覆盤報告：\n\n" + aiReport;
    UrlFetchApp.fetch("https://api.telegram.org/bot" + telegram_token + "/sendMessage", {
      "method": "post", "contentType": "application/json", 
      "payload": JSON.stringify({ "chat_id": telegram_chatId, "text": message })
    });
  } catch (e) {
    console.log("夜間覆盤執行失敗：" + e);
  }
}

// ==========================================
function scanPotentialStocks() {
  // 🛡️ 1. 將讀取金鑰直接寫在函式內，保證絕對不會「not defined」
  var scriptProperties = PropertiesService.getScriptProperties();
  var telegram_token = scriptProperties.getProperty('TG_TOKEN');
  var telegram_chatId = scriptProperties.getProperty('TG_CHAT_ID');
  var gemini_apiKey = scriptProperties.getProperty('GEMINI_KEY');

  // 2. 讀取您的專屬追蹤清單
  var ss = SpreadsheetApp.getActiveSpreadsheet();
  var trackSheet = ss.getSheetByName("追蹤清單");

  var trackData = trackSheet.getDataRange().getValues();
  var myStocks = [];
  
  // 從第 2 列開始抓取 (避開標題列)
  for (var i = 1; i < trackData.length; i++) {
    var code = trackData[i][0].toString().replace("TPE:", "").trim(); // 抓代號
    var name = trackData[i][1].toString().trim();                     // 抓名稱
    if (code !== "") {
      myStocks.push(name + "(" + code + ")");
    }
  }

  // 如果清單是空的，就不執行
  if (myStocks.length === 0) return;
  
  var stockNames = myStocks.join("、");
  var aiAnalysis = "AI 分析處理中...";

  // 3. 專屬健檢 Prompt
  var prompt = `你是頂尖台股波段與動能操盤手。這是今日的專屬持股健檢標的：${stockNames}。
請針對標的進行「中短期操作」的盤後量價與潛力分析，並嚴格遵循以下 4 大板塊進行綜合研判。

⚠️【嚴格輸出規範】：
1. 保持客觀冷靜、語氣果斷，絕對去除所有寒暄、結語與免責聲明。

【一、市場位階與動能】
* 市場熱度：判斷當前資金動能，是否出現異常爆量或量價背離。
* 相對強弱：對比大盤，確認為資金避風港、強勢領頭羊或領跌指標。
* 型態與均線：目前處於突破發動、量縮回檔或高檔轉弱？（需指出與 5日/10日/月線之關聯）。

【二、籌碼與基本面催化】
* 法人籌碼：外資與投信近期動向（連續點火 / 高檔換手 / 結帳出貨）。
* 近期催化劑：即將發布的法說會、營收、除權息等重大消息面。
* 產業與族群：產業基本面速覽，有無同族群題材連動效應。

【三、實戰劇本與進場策略】
* 支撐與壓力：精準標示近期關鍵壓力區間與核心防守價位。
* 最佳進場劇本：設定當下最適用的進場情境（如：突破ＯＯ進場 / 拉回ＯＯ量縮佈局）及其背後邏輯。

* 停損防守：給出明確撤退紀律（如：跌破某關鍵K線低點或某均線即刻出場）。
* 減碼策略：走勢不如預期或高檔爆量時的具體減碼條件。
* 損益比(R/R)試算：以建議進場價為基準，計算至上方壓力(獲利)與下方防守(停損)之比例。⚠️ 若損益比 < 2，請務必在此項標示【風險過高警示】。

⚠️ 絕對要求：
請在整篇分析的最末端，為每檔股票加上這一行格式化的數據（代號、支撐、壓力、策略之間請使用「|」符號分隔，策略請濃縮在 100 字以內,以淺顯易懂的說明方式，依照我的專屬追蹤清單標的換行）：

例如：[DATA:2330|1050|1120|量縮回測支撐不破可低接]

回覆內容請一律使用『純文字』，絕對不要使用任何 Markdown 格式符號。`;

  var geminiUrl = "https://generativelanguage.googleapis.com/v1beta/models/gemini-flash-latest:generateContent?key=" + gemini_apiKey;

        "method": "post", 
        "contentType": "application/json", 
        "payload": JSON.stringify(payload),
        "muteHttpExceptions": true
      });
      
      var responseCode = aiResponse.getResponseCode();
      
      if (responseCode === 200) {

        aiAnalysis = aiResult.candidates[0].content.parts[0].text;
        success = true;
        break; 
      } else {
        Utilities.sleep(3000); 
      }
    } catch(e) {
      Utilities.sleep(3000);

  }

  // 5. 解析結果與回填試算表
  if (!success) {

    while ((match = regex.exec(aiAnalysis)) !== null) {
      var stockCode = match[1];     
      var supportPrice = match[2];  
      var resistancePrice = match[3]; 
      var strategy = match[4];      
      
      for (var i = 1; i < trackData.length; i++) {

  if (logSheet) logSheet.appendRow([new Date(), "專屬持股健檢", aiAnalysis]);

  var message = "🏥 【盤後專屬持股健檢】\n" +
                "🎯 健檢對象：追蹤清單標的\n" +
                "━━━━━━━━━━━━━━━\n\n" + 
                aiAnalysis + "\n\n" +
                "📌 系統狀態：" + (success ? "✅ 壓力、支撐與【隔日策略】已同步至清單。" : "❌ 伺服器忙碌，請稍後重試。");

  UrlFetchApp.fetch("https://api.telegram.org/bot" + telegram_token + "/sendMessage", {
    "method": "post", "contentType": "application/json", 
    "payload": JSON.stringify({ "chat_id": telegram_chatId, "text": message })
  });
}
```

## 五大齒輪詳解

### 齒輪一：`fetchStockData()` — 每日抓取收盤

**資料來源**：證交所 TWSE OpenAPI（免費、無需金鑰）

```
https://openapi.twse.com.tw/v1/exchangeReport/STOCK_DAY_ALL
```

**核心邏輯**：
1. 抓取全市場所有股票當日收盤資料（JSON 格式）
2. 過濾出 `myStocks` 陣列中的自選股代號
3. 將日期、代號、名稱、收盤價、漲跌、成交量寫入「每日行情」分頁

**自訂自選股**：修改 `myStocks` 陣列即可：
```javascript
var myStocks = ["2330", "2317", "0050"]; // 填入你要追蹤的股票代號
```

---

### 齒輪二：`nightlyReview()` — 夜間覆盤報告

**執行時機**：盤後（建議晚上 9 點）

**流程**：
1. 從「每日行情」讀取當日最新資料
2. 整理成「股名+漲跌」的摘要字串
3. 送入 Gemini，要求生成 **150 字內** 的純文字覆盤報告
4. 報告存入「AI紀錄」並推播至 Telegram

**Prompt 設計重點**：
- 角色設定：「頂尖台股操盤手」
- 輸出格式：要求純文字，禁用 Markdown（避免 Telegram 顯示亂碼）
- 內容要求：整體盤勢簡評 + 隔日建議

---

### 齒輪三：`morningForecast()` — 晨間作戰計畫

**執行時機**：開盤前（建議早上 8 點）

**資料來源**：「國際盤勢」分頁的 `A2:B4`（需手動輸入前一晚美股數據）

**Prompt 涵蓋 15 大分析面向**：
1. 市場熱度與量價結構
2. 技術面支撐與壓力
3. 操作策略與風控
4. 近期催化劑與產業速覽
5. 股價位階與大盤強弱對比
6. 主力與三大法人籌碼
7. 法說會、營收、除權息等重大事件
8. 量價型態與均線位階
9. 投信外資籌碼研判
10. 實戰劇本與進場情境
11. 嚴格停損條件
12. 同族群題材連動分析
13. 分批布局區間或突破觸發點
14. 進場邏輯說明
15. 損益比（R/R）試算

---

### 齒輪四：`generateAnalysisDoc()` — 深度分析文案

**用途**：針對自選股生成較完整的書面分析報告

**與齒輪二的差異**：
- 齒輪二：快速覆盤，150 字，給結論
- 齒輪四：完整分析，含明日風險提示，存入「AI紀錄」備查

---

### 齒輪五：`scanPotentialStocks()` — 盤後持股健檢（核心功能）

**最強大的齒輪**，具備以下特點：

#### 自帶金鑰機制
在函式內部重新讀取一次 Script Properties，防止全域變數失效導致 `not defined` 錯誤。

#### 動態讀取追蹤清單
自動從「追蹤清單」分頁抓取所有股票代號與名稱，支援 `TPE:` 前綴格式（TradingView 格式）。

#### 四大分析板塊 Prompt
| 板塊 | 內容 |
|------|------|
| 一、市場位階與動能 | 熱度、相對強弱、均線型態 |
| 二、籌碼與基本面催化 | 法人動向、催化劑、族群效應 |
| 三、實戰劇本與進場策略 | 支撐壓力、進場劇本、觸發價位 |
| 四、嚴格風控與期望值 | 停損、減碼、R/R 損益比試算 |

#### 結構化資料解析（DATA 標籤）
Prompt 要求 AI 在報告末尾輸出特定格式：
```
[DATA:代號|支撐價|壓力價|隔日策略]
範例：[DATA:2330|1050|1120|量縮回測支撐不破可低接]
```

程式以正規表達式解析後，**自動回填**至「追蹤清單」對應欄位：
- C 欄 ← 支撐價
- D 欄 ← 壓力價
- H 欄 ← 隔日策略

#### 自動重試機制
最多重試 3 次，每次間隔 3 秒，應對 Gemini API 503 壅塞。

### 為何不把金鑰寫在程式碼裡？

| 做法 | 風險 |
|------|------|
| 直接寫在程式碼 | 若分享給他人或公開專案，金鑰立即外洩 |
| 使用 Script Properties | 金鑰加密儲存，程式碼可安全分享 |

### Script Properties 讀取方式
```javascript
var scriptProperties = PropertiesService.getScriptProperties();
var myKey = scriptProperties.getProperty('MY_KEY_NAME');
```

## 常見問題排解

### Q1：Telegram 沒有收到訊息
- 確認 Bot Token 與 Chat ID 正確
- 確認 Bot 已被加入聊天室且未被封鎖
- 檢查 Apps Script 執行紀錄是否有錯誤（左側「執行項目」）

### Q2：Gemini 回傳 503 錯誤
- 為正常現象，系統會自動重試 3 次
- 若持續失敗，表示 API 用量超過免費額度上限
- 確認 `gemini-flash-latest` 模型名稱是否正確

### Q3：`fetchStockData` 沒有寫入資料
- 確認工作表名稱為「每日行情」（含中文，全形符號）
- 確認 `myStocks` 陣列內的代號為字串格式
- 注意：TWSE API 只在交易日有資料，假日執行會得到空回應

### Q4：`scanPotentialStocks` 找不到追蹤清單
- 確認工作表名稱為「追蹤清單」
- 確認第一列為標題列，資料從第二列開始
- 代號欄支援 `2330` 或 `TPE:2330` 兩種格式
